## What happened

On the 2026-05-22 15:05 reppo-swarm run, the orchestrator emitted `RUN`
for datanet 9 (TradingGym AI, ACTIVE & valid) and the trading agent queued
4 intents:

- `mint-122bce0dc8111ad1.json` — BTC/USD 5m EMA-crossover momentum strategy
- `vote-257-like.json`
- `vote-332-dislike.json`
- `vote-46-dislike.json`

`scripts/postprocess-reppo.sh` then ran in `dry_run_only=true` mode. Its
appended `## Execution Results` section reports that **all four** intents
failed their dry-run:

```
mint-122bce0dc8111ad1.json — dry-run failed (code: UNKNOWN), real write skipped
vote-257-like.json        — dry-run failed (code: UNKNOWN), real write skipped
vote-332-dislike.json     — dry-run failed (code: UNKNOWN), real write skipped
vote-46-dislike.json      — dry-run failed (code: UNKNOWN), real write skipped
```

## Recurrence

- **2026-05-22 (later run)** — reproduced. Orchestrator emitted `RUN` for
  datanet 9; the trading agent queued `mint-81fa801019caf579.json` (crypto
  perpetual funding-rate carry) plus `vote-332/46/300-dislike.json`.
  `postprocess-reppo.sh` again ran `dry_run_only=true` and all four intents
  failed dry-run with `code: UNKNOWN`, real writes skipped. Same failure
  mode, different intents — confirms this is not intent-specific.

## Impact

- 0 strategies minted, 0 votes cast on-chain — the chain produced no
  productive on-chain output this run.
- This is a **distinct failure mode from ISS-002**. ISS-002 was the
  placeholder `datanet_id` blocking the orchestrator before any intent was
  generated; here the orchestrator resolved datanet 9 and intents were
  generated and queued correctly. The failure is now downstream, inside
  postprocess dry-run validation.
- Earlier 2026-05-22 runs had postprocess **skip** all intents because
  `REPPO_PRIVATE_KEY` was unset. This run instead **attempted** dry-runs and
  they **failed** with `code: UNKNOWN` — a new, uncharacterized error.

## Fix

Investigate why `postprocess-reppo.sh` dry-runs return `code: UNKNOWN`.
The `UNKNOWN` code suggests the Reppo CLI/API returned an error the script
could not classify. Likely lines of inquiry: capture and surface the raw
CLI/API response (don't collapse it to `UNKNOWN`), verify the Reppo CLI
version and endpoint are current, and confirm the intent JSON schema still
matches what the CLI expects. Until the dry-run passes, no real write will
ever be attempted even with a private key configured.
