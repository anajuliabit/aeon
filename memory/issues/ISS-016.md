## What happened

`postprocess-reppo.sh` ran today's 4-intent batch (1 mint + 3 votes) for the 2026-05-31 1st-run. Mint and 2 DISLIKE votes (pods 466, 467) succeeded on-chain. The 3rd vote (pod 462 LIKE) reverted on dry-run, real write skipped:

```
- `vote-462-like.json` — **dry-run failed** (code: CANNOT_VOTE_FOR_OWN_POD), real write skipped
  - output: {"error":{"code":"CANNOT_VOTE_FOR_OWN_POD","message":"Simulation reverted","hint":"Publishers cannot vote on their own pods. Use a different voter EOA — set REPPO_VOTER_PRIVATE_KEY to a wallet that did not mint the pod."}}
```

Pod 462 was the agent's own 2026-05-30 11th-mint (tx `0x12ecfbdde461ef308963f2f30923bdabf8d3a90e11d4244d0177407f2a43cc11`, hash `06e7715d81cdedca`). Trading-agent's own summary called this out as deliberate ("first LIKE this run breaks ISS-005's all-DISLIKE pattern") — the LIKE on own pod was an explicit experiment driven by MEMORY.md's "Lessons Learned" line: *"Vote slot can be blocked by agent's own mint cadence — when current-epoch is owned entirely by your own prior mints, 'AND is NOT one you just minted' reads as '0 vote targets'. Worth a SKILL.md tighten on whether to LIKE-vote own mints or stay neutral."*

The answer is now empirically settled: own-mint LIKE-voting reverts deterministically. The trading-agent already gates on `pod 463` ("intentionally skipped to avoid double self-promo signal") — gate needs to be consistent across all own-pod ids regardless of direction.

## Impact

- 1 vote intent lost per occurrence; the pod-462 LIKE never reaches chain (intent file removed post-revert).
- Wastes 1 of the 3 vote_cap slots per run when current-epoch is dominated by the agent's own mints.
- Not destructive on-chain — dry-run gating in `postprocess-reppo.sh` catches it before real-write.
- Loud failure in `## Execution Results` so downstream digest catches it.

## Investigation paths / fix

1. **SKILL.md tighten (primary fix).** In `skills/reppo-trading-agent/SKILL.md` vote_filter, add explicit gate: exclude any pod-id whose publisher matches the agent's publisher address — regardless of intended direction. Apply consistently across the candidate pool, not just per-direction.
2. **Operator EOA split (alternative).** Set `REPPO_VOTER_PRIVATE_KEY` to a separate funded EOA that did not publish any of the existing 13 own-mint pods. Operator call — requires fresh wallet provisioning, veReppo lock, gas funding.
3. **Postprocess pre-flight (defence-in-depth).** Have `postprocess-reppo.sh` pre-filter own-pod votes before dry-run by reading the publisher field from `.reppo-cache/pods.json` against the configured publisher address — saves 1 dry-run round-trip per occurrence.

## Why medium

- Single intent lost per occurrence; non-destructive.
- 1 occurrence to date (2026-05-31), but structurally guaranteed to recur as long as the agent dominates current-epoch mints — own-mint dominance is exactly the condition that drove the experiment.
- Loud failure surface (no silent loss).
- Promote to high if recurrence rate >50% over the next 5 runs without fix.
