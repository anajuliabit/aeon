## What happened

vibecoding-digest's 3rd consecutive run with all 3 sources (top/hot/rising) failing:

- prefetch: all 3 cache files contain `{"code":"PREFETCH_FAILED","error":"fetch <name>.json failed"}` — host-side curl from the GitHub Actions runner is being rejected by Reddit, despite the host running outside the inner sandbox.
- WebFetch fallback per skill step 1: returns "Claude Code is unable to fetch from old.reddit.com" / "www.reddit.com" for all 3 endpoints. Same response for the bare `reddit.com` host.

Result: 0 scanned, 0 new, 0 dedup'd. Notification staged with `VIBECODING_DIGEST_ERROR`.

## Why it's filing-worthy now

- 2026-05-28 1st failure (logged).
- 2026-05-29 2nd failure (logged; closing line: "Filing-worthy if it fails a 3rd time tomorrow").
- 2026-05-30 3rd consecutive failure — same root cause, same surface.

Per skill spec, the dup-notify suppression already fires on same-day 2nd-runs (added in PR #43). The cross-day pattern is what surfaces as an issue.

## Why high

- The skill has zero data path. It is structurally broken, not flaky.
- 3 consecutive runs over 3 days, no organic recovery.
- The fallback path the skill spec defines (WebFetch) is itself blocked at the tool layer — there is no in-skill workaround left.

## Investigation paths

1. **Confirm host-side block.** SSH-equivalent debug run from `scripts/prefetch-vibecoding.sh` — log the actual HTTP status + response body Reddit returns. If GitHub Actions runner IPs are now broadly rejected (not just the sandbox), the prefetch pattern is dead for Reddit.
2. **Check the WebFetch deny path.** WebFetch returning "Claude Code is unable to fetch from <host>" suggests an explicit deny list at the tool layer, not a generic network error. Confirm whether reddit.com is on the Claude Code WebFetch deny list, and whether that's recent or permanent.
3. **Alternative sources** if both above are confirmed dead:
   - **Authed Reddit API** (`oauth.reddit.com`) — requires a Reddit app + `REDDIT_CLIENT_ID` / `REDDIT_CLIENT_SECRET` repo secrets. Bearer-token requests are less aggressively rate-limited and often route through different infra than anonymous JSON endpoints.
   - **Pushshift / Arctic Shift archive** — 24h+ lag, but the digest already runs once daily; freshness loss is small.
   - **HN /r/programming mirror** — broader scope than vibecoding-specific, drops the subreddit-niche angle entirely.

## Notes

- `scripts/prefetch-vibecoding.sh` already documents the sandbox block in its own comments. The new fact today is that the WebFetch *fallback* is also dead — that breaks the skill's intended degradation path.
- PR #43 (dup-notify suppression) is the right same-day mitigation but doesn't address the structural data-source loss.
- Filing under `sandbox-limitation` rather than `api-change` because Reddit's behavior hasn't changed — the issue is the runner/sandbox/tool-layer combination losing all paths to it.
