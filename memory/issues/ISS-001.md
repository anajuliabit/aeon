# ISS-001: agent-buzz has no prefetch case

## Summary
The `agent-buzz` skill depends on X.AI `x_search` (Grok) to fetch fresh AI-agent
tweets with engagement metrics. In the GitHub Actions sandbox, an authenticated
`curl` POST to `https://api.x.ai/v1/responses` cannot run (env-var expansion
blocked / requires approval). The intended workaround is the pre-fetch pattern in
`scripts/prefetch-xai.sh`, but that script has **no `agent-buzz` case**, so
`.xai-cache/` is empty when the skill runs.

## Impact
- Step 1 fallback chain fails through:
  1. curl to X.AI — blocked by sandbox.
  2. WebFetch the X.AI endpoint — cannot issue an authenticated POST.
  3. WebSearch — succeeds as a call but returns only news articles and stale
     (>48h) tweets; no fresh, datable tweets with engagement counts, so signal
     scoring (step 3) and skip-gates (step 2) cannot run honestly.
- Result on 2026-05-21: `AGENT_BUZZ_ERROR` — no digest produced.

## Fix
Add an `agent-buzz` case to `scripts/prefetch-xai.sh` modeled on the existing
`fetch-tweets` / `narrative-tracker` cases. It should issue the step-1
`x_search` query (AI-agents conversation, yesterday→today, ~40 candidates with
@handle/follower/role/claim/engagement/posted_at/direct_link) and save to
`.xai-cache/agent-buzz.json`. The skill's step 1 should then read that cache
before attempting curl.

Same class of gap was already noted for the `last30` skill (see
`memory/logs/2026-05-21.md` — "X in missing-prefetch mode").
